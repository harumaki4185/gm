# リーチ麻雀 実装仕様書

## 1. 目的

本書は、Classic Duels に 4 人打ちリーチ麻雀を追加し、通常対局を最後まで成立させるための実装仕様を定義する。
対象は Cloudflare Workers + Durable Objects 上で動作する、招待リンク型・ログイン不要のオンライン対戦麻雀である。

この仕様でいう「全部対応」は、少なくとも以下を指す。

- 4 人打ちリーチ麻雀の通常進行を 1 局目から終局まで完走できる
- bot を含む 1-4 人構成で対局を成立させられる
- 鳴き、立直、和了、流局、点数計算、場進行、順位計算をサーバー側で完結できる
- 非公開情報をプレイヤーごとに分離したまま、観戦・再接続にも対応できる

## 2. 対象ルール

### 2.1 基本ルール

- 4 人打ち日本式リーチ麻雀
- 牌山 136 枚
- 1 ルーム = 1 卓
- 人間 1-4 人 + 不足分 bot
- 招待リンク型ルーム
- ログイン機能なし

### 2.2 対局形式

- 東風戦
- 半荘戦

### 2.3 初期ルール設定

初期実装では以下をデフォルトとする。

- 喰いタン: あり
- 赤ドラ: あり
  - 5m / 5p / 5s を各 1 枚
- 裏ドラ: あり
- 槓ドラ / 槓裏ドラ: あり
- ダブロン: あり
- トリプルロン: 流局
- 飛び終了: あり
- 延長戦:
  - 東風戦は東 4 局終了時点で終了
  - 半荘戦は南 4 局終了時点で終了
  - オーラス終了時にトップ同点は起家寄り優先
- ウマ / オカ: なし
  - 終局順位は素点順位のみで決定

### 2.4 ルールオプション

将来拡張を見据えて、以下は room settings として予約する。

- `matchType`: `tonpuu` | `hanchan`
- `akaDora`: boolean
- `kuitan`: boolean
- `doubleRon`: boolean
- `tripleRonMode`: `abortive_draw` | `triple_ron`
- `tobiEnd`: boolean
- `uma`: number[] | null
- `oka`: number | null

## 6. 進行仕様

### 6.1 対局開始

- 開始時に席順をランダム決定する
- 起家をランダム決定する
- 東家を 0 として時計回りに
  - 東
  - 南
  - 西
  - 北
- 対局形式に応じて場を開始する

### 6.2 局開始

- 牌山をシャッフルする
- 王牌 14 枚を確保する
- 各プレイヤーへ 13 枚配牌
- 親のみ 14 枚スタート
- ドラ表示牌を 1 枚公開
- 河 / 副露 / 立直状態 / 一発状態 / フリテン状態を初期化

### 6.3 通常ターン

1. 自摸
2. 自摸直後行動判定
   - ツモ和了
   - 暗槓
   - 加槓
   - 立直可能判定
3. 打牌
4. 他家の反応待ち
   - ロン
   - ポン
   - 大明槓
   - チー
   - パス
5. 割り込み解決
6. 次手番へ進行

### 6.4 他家反応優先順位

優先順位は以下とする。

1. ロン
2. ポン / 大明槓
3. チー

同優先度で複数候補がある場合:

- ロンは複数和了を許可する
  - ダブロン対応
  - トリプルロンは設定に従う
- ポン / 大明槓は打牌者から近い順
- チーは上家のみ

### 6.5 立直

- 門前手のみ可能
- 1000 点以上所持していること
- 立直後は手牌固定
  - 暗槓可否は待ちが変わらないケースのみ
- 宣言牌確定後に供託を積む
- 一発フラグは以下で消える
  - 他家副露
  - 槓成立
  - 自身次巡までに和了しなかった

### 6.6 フリテン

以下を区別して保持する。

- 永続フリテン
  - 自身の河に和了牌がある
- 同巡フリテン
  - 同巡内でロン可能牌を見逃した
- リーチ後フリテン
  - リーチ後にロン可能牌を見逃した

### 6.7 槓

対応する槓種:

- 暗槓
- 大明槓
- 加槓

処理:

- 槓成立後に嶺上牌を補充
- 槓ドラ表示牌を追加公開
- 裏ドラは和了時に必要枚数分だけ参照
- 四開槓流局条件は場全体の槓数で判定する

### 6.8 流局

荒牌平局時:

- テンパイ / ノーテンを判定
- ノーテン罰符を処理
- 親テンパイなら連荘

途中流局時:

- 理由を局結果として残す
- 供託は持ち越す
- 本場は加算する

### 6.9 終局条件

- 東風戦:
  - 東 4 局終了時点で終了
- 半荘戦:
  - 南 4 局終了時点で終了
- 飛び終了が有効で 0 点未満が発生した場合は終了

### 7.2 加算要素

## 8. bot 仕様

### 8.1 対応範囲

bot は以下をすべて行えること。

- 配牌評価
- 打牌選択
- リーチ判断
- ロン / ツモ判断
- チー / ポン / 槓判断
- オリ判断

### 8.2 最低限の思考要件

- シャンテン数を計算できる
- 受け入れ枚数を比較できる
- 安牌候補を認識できる
- 立直者がいる場合は防御寄りに切り替える
- 和了可能時は必ず和了する

### 8.3 初期実装の強さ

初期の目標は「成立する」「明らかな違法手を打たない」であり、最強 AI は目指さない。

- 難易度は 1 種類から開始
- 後から `easy / normal / hard` へ拡張できる設計にする

## 9. 観戦 / 再接続仕様

### 9.1 再接続

- 同一 sessionId で卓へ復帰可能
- 手牌、待機中選択、局情報を復元する
- 行動待ち中に復帰した場合は残り時間も復元する

### 9.2 観戦

- 観戦者は非公開情報を見られない
- 手牌は枚数のみ表示
- 山牌順 / 王牌内部 / 待ち牌は非表示
- リプレイ用途の完全情報は公開しない

## 10. サーバー権威モデル

- すべての牌順は Worker 側で決定する
- クライアントは合法行動の要求だけを送る
- 和了判定、鳴き判定、点数計算はすべて Worker 側で行う
- クライアントは点数や役を計算しない
- 非公開情報は player ごとに view を変えて返す

## 13. Worker 実装方針

### 13.1 追加モジュール案

- `worker/games/mahjong/index.ts`
- `worker/games/mahjong/types.ts`
- `worker/games/mahjong/flow.ts`
- `worker/games/mahjong/legal.ts`
- `worker/games/mahjong/yaku.ts`
- `worker/games/mahjong/scoring.ts`
- `worker/games/mahjong/shanten.ts`
- `worker/games/mahjong/bot.ts`
- `worker/games/mahjong/view.ts`
- `worker/games/mahjong/tiles.ts`

### 13.2 設計原則

- 牌は文字列ではなく一意な tile id で管理する
- UI 表示用の牌種変換は view 層でのみ行う
- 乱数は対局開始時の seed を持ち、必要なら再現可能にする
- `responseWindow` を明示状態として保持する
- 局結果の点数計算は pure function で切り出す

### 13.3 Durable Objects 上の制約

- 1 卓の操作は単一 Durable Object で直列処理する
- タイマーは alarm で管理する
- 行動期限切れ時は以下を自動処理する
  - 自手番: ツモ切りまたは pass
  - 割り込み待ち: pass

## 14. UI 実装方針

### 14.1 PC

- 下家を手前固定
- 他家は上 / 左 / 右
- 手牌を常に下部表示
- 河は家ごとに独立表示
- 副露は牌向きでチー / ポン / 槓元を示す

### 14.2 スマートフォン

- 自手牌は横スクロール対応
- 他家情報は圧縮表示
- 行動ボタンはモーダルまたは固定バー表示
- 長考不要の操作を優先し、詳細情報は展開 UI に逃がす

### 14.3 結果表示

- 局終了ごとにモーダル表示
- 対局終了時は順位表へ遷移
- 役一覧は略称ではなく日本語名で表示

## 17. 受け入れ条件

- 人間 1-4 人構成で麻雀対局を開始できる
- bot を含む卓が終局まで停止せず進行する
- 鳴きの優先順位が仕様どおりに処理される
- 和了時に役 / 翻 / 符 / 点数が正しく表示される
- 荒牌流局と途中流局が正しく処理される
- 本場 / 供託 / 連荘 / 親流れが正しく進行する
- 再接続後に手牌と局状態が復元される
- 観戦者に手牌と山牌情報が露出しない
- 対局終了時に最終順位が表示される
