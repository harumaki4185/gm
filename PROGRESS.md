# Progress

## 現在の実装状況

### 基盤

- Cloudflare Workers + Durable Objects の構成を作成
- React + TypeScript のフロントエンドを作成
- SPA ルーティング用の軽量ルーターを追加
- 招待リンクによるルーム作成、参加、再接続、再戦を実装
- WebSocket によるリアルタイム同期を実装

### セキュリティ / 安全性

- `rematchVotes` から `sessionId` を排除
- ルーム ID を UUID ベースへ変更
- `POST /api/rooms` に Durable Object ベースのレート制限を追加
- 表示名に禁止ワード判定と制御文字除去を追加
- 切断時の不戦勝処理と待機ルーム自動破棄を追加

## 今回追加した内容

### 可変人数対応

- ゲーム定義に `defaultSeats / minSeats / maxSeats` を追加
- ルーム設定に `seatCount` を追加し、ゲームごとに人数を指定できるように変更
- ゲーム詳細画面から人数と bot 補充有無を選んでルーム作成できるように変更
- 進行中ルームへの新規参加を禁止し、待機中のみ参加できるように変更

### 構造整理

- Worker 側のゲームロジックを `worker/games/` 配下へ分割
- フロント側のゲーム画面を `src/components/games/` 配下へ分割
- `index.ts` / `GameSurface.tsx` を集約ルーターに寄せ、ゲーム別実装を独立させた
- `src/shared/cards.ts` を追加し、カード生成・シャッフル・並び替え・表示を共通化した

### じゃんけん

- 2-6 人対応へ拡張
- 複数勝者判定を追加
- 3 手すべてが出た場合や全員同手の場合をあいことして次ラウンドへ進めるように変更
- 勝者を UI 上で強調表示するように変更

### ババ抜き

- 2-4 人対応へ拡張
- Worker 側に巡回ターン、前席からのドロー、複数勝者判定を追加
- bot 手番を Worker 側で自動進行するように変更
- プレイヤーごとに異なる表示状態を返すビューを多人数向けに更新
- フロント側に `引く相手` を明示した複数相手 UI を追加
- 自分の手札表示、直前アクション表示、勝敗表示を多人数向けに更新

### 七並べ

- 2-4 人対応で実装
- 4 スートの 7 を自動配置し、隣接カードのみ出せるルールを実装
- 出せるカードがない場合のみパスできるようにした
- bot 手番を Worker 側で自動進行するようにした
- 場の進行状況と手札を表示する専用 UI を追加した

### スペード

- 4 席固定の 2 対 2 ペア戦として実装
- ビッドフェーズとトリックフェーズを Worker 側で管理するようにした
- フォロー義務、スペード解禁、トリック勝者判定、1 ハンド得点計算を実装
- bot のビッドとカード選択を Worker 側で自動進行するようにした
- チーム状況、現在トリック、手札操作を表示する専用 UI を追加した

### 補助画面

- ゲーム詳細ページを追加
- ルール / ヘルプ画面を追加
