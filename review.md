### 🟡 Minor

#### m-V14-1 牌 ID の重複リスクについて（設計の注意点）

**ファイル:** `src/shared/mahjong.ts:14`

牌 ID の形式は `m1-0`, `m1-1`, `m1-2`, `m1-3` のように `種類-コピー番号` で一意性を担保しています。`discard` 処理で牌を `hand.indexOf(tile)` で検索しているため、同じ種類の牌が複数枚あっても正しく先頭1枚を取り除きます（`-0`, `-1` で区別されているため）。

ただし、大明槓・加槓などを将来実装する際に「4枚すべての ID を追跡する必要がある（`indexOf` だと1枚しか消えない）」という点に注意が必要です。Phase 1 の範囲内では現在問題ありません。

#### m-V14-2 カタログの `category: "table"` が `GameCatalogEntry` 型に合わない

**ファイル:** `src/shared/games.ts:107`

```ts
category: "table",  // ← 現行型は "board" | "card" | "party" のみ
```

`GameCatalogEntry.category` の型に `"table"` が含まれていません。TypeScript のコンパイラが型チェックをしていれば型エラーになります（ランタイムには影響なし）。`"card"` または `"table"` を型に追加するどちらかで対応してください。
